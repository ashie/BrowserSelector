BrowserSelector設定仕様について（v1.2.0）

設定は以下の順番で読み込まれます。
後に読み込まれるものの方が設定の優先度が高くなります。

  * HKLMレジストリ
    * HKEY_LOCAL_MACHINE\SOFTWARE\WOW6432Node\ClearCode\BrowserSelector
  * インストールフォルダのiniファイル
    * 例: C:\Program Files (x86)\ClearCode\BrowserSelector\BrowserSelector.ini
    * デフォルトでは上記ファイルは存在しません。手動で設置する必要があります。
  * HKCUレジストリ
    * HKEY_CURRENT_USER\SOFTWARE\WOW6432Node\ClearCode\BrowserSelector
  * ユーザーフォルダのAppData内のiniファイル
    * 例: C:\Users\[UserName]\AppData\Roaming\ClearCode\BrowserSelector\BrowserSelectror.ini
    * デフォルトでは上記フォルダやファイルは存在しません。手動で設置する必要があります。

また、上記いずれかのiniファイルから外部iniファイルのパスを指定して設定をマージすることができます。
マージされた外部iniファイルからさらに外部iniファイルをマージすることはできません。

iniファイルの設定内容は以下の通りです。
設定例はsample以下のファイルを参照して下さい。

  * [Common]セクション
    * DefaultBrowser: URLパターンやホスト名にマッチしなかった場合に使用されるブラウザ
      * 以下のいずれかを指定
        * ie (デフォルト)
        * chrome
        * firefox
    * SecondBrowser: URLやホスト名パターンでブラウザ名が指定されていない場合に使用するブラウザ
      * 以下のいずれかを指定
        * ie
        * chrome
        * firefox
      * 指定しない場合はDefaultBrowserを使用
    * CloseEmptyTab: 外部ブラウザ起動後に残された空タブを閉じるか否かの設定
      * 1: 閉じる (デフォルト)
      * 0: 閉じない
    * UseRegex: URLやホスト名のパターンマッチングにおいて、正規表現を使えるようにするか否かの設定
      * 以下のいずれかを指定
      * 0: 正規表現を使用せず、「*」および「?」による単純なワイルドカードによるマッチングを行う（デフォルト）
      * 1: 正規表現を使用する（文法については https://ja.cppreference.com/w/cpp/regex/ecmascript を参照）
    * OnlyOnAnchorClick: リンクのクリックのみをブラウザ切り替えの対象とするか否かの設定
      * 0: すべてのページ遷移をブラウザ切り替えの対象とする（デフォルト）
      * 1: リンクのクリックのみをブラウザ切り替えの対象とする
    * Include: 外部iniファイルを指定
      * 以下のいずれかの形式を使用可能
        * 通常のWindowsパス形式: C:\path\to\file.ini
        * UNC形式: \\HostName\ShareName\path\to\file.ini
    * EnableIncludeCache: 外部iniファイルにアクセスできない場合にキャッシュを使用するかどうかを指定
      * 0: 使用しない（デフォルト）
      * 1: 使用する
      * キャッシュはユーザーフォルダのLocalAppDataLow以下に保存される
        * 例: C:\Users\[UserName]\AppData\LocalLow

  * [URLPatterns]セクション
    * URLパターンと、そのパターンを開くブラウザ名のリストを key=url|browsername の形式で記述
    * keyは任意の文字列です。keyとして認識できる文字列で他と重複しなければ何でも構いません。
    * 値はURLパターンとブラウザ名の間を「|」で区切って下さい。
    * デフォルトではURLパターンには以下の2種類のワイルドカードを使用できます。
      * 「*」: すべての文字列にマッチ
      * 「?」: 任意の位置文字にマッチ
    * SecondBrowserの設定を使う場合、「|」とブラウザ名は省略することができます。
    * UseRegex=1をセットすると、上記の代わりに正規表現が使えるようになります。
      * 正規表現の文法については以下を参照して下さい
        * https://ja.cppreference.com/w/cpp/regex/ecmascript
      * 正規表現を使う場合、ブラウザ名の指定は「|」ではなく行末アサーション「$」の後に記述してください。
        * 例: 0001=^http(s)?://(.+\\.)?example\\.(org|com)(/.*)?$firefox

  * [HostNamePatterns]
    * URLPatternsとほぼ同様ですが、URLのホスト名部分のみでマッチングします
    * URLPatternsとHostNamePatternsの両方が設定されている場合、URLPatternsの方が優先されます。

レジストリで設定できる内容はiniファイルとほぼ同様ですが、Includeは使用できません。
